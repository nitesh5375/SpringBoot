What is the difference between Stateful authentication (Session-based) and Stateless authentication (JWT)?
Why do modern systems prefer JWT?**

ANSWER:
Stateful Authentication (Session-based)
Server stores session data (user identity, roles, etc.)
Client sends session ID (cookie) on each request
Server uses that ID to fetch user info from memory / Redis / DB

Stateless Authentication (JWT-based)
Server does not store session
JWT itself contains:
user identity
roles

Client sends JWT with every request
Server only validates token, no lookup required
